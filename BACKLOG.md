## A. W0 合规与产品验证（🔴 长板，第 0 周就启动，和开发并行）

**合规（需属地法律意见，多数非编码、有法定周期）**
- [ ] 🔴 P0 提交 **ICP 备案**（法定 ~20 工作日，决定能否上线，最先做）
- [ ] 🔴 P0 出**数据流清单**：收集项/目的/位置/处理者/保留期/删除方式
- [ ] 🔴 P0 做 **PIPIA**（个人信息保护影响评估）并留档（记录存 ≥3 年）
- [ ] 🔴 P0 隐私政策 + 用户协议 + **上传单独同意**文案（含"本人或已授权"声明）
- [ ] 🔴 P0 与 LLM/云/短信厂商签 **DPA**（目的/期限/数据种类/安全/删除/转委托/国内地域/不训练）
- [ ] P0 **等保定级**判断 + 负责人制度 + 应急预案（网络日志留存 ≥6 个月）
- [ ] P0 **AI 生成内容标识**方案（显式 + 元数据隐式；与商业水印分离）
- [ ] P0 **生成式 AI 应用登记** + 公示所用模型及备案号；确认是否涉经营性 ICP 许可

**产品验证（决定生死，和开发并行）**
- [ ] 🔴 P0 15–20 次目标用户访谈，定 **1 个垂类**（文本密集型：产品/运营/数据，非设计师）
- [ ] 🔴 P0 定**单一获客渠道**（小红书/校招中心/垂直社群）
- [ ] P0 建 **50 份脱敏真实简历金标集**（评估解析/匹配/改写错误率，作上线质量门）
- [ ] P1 招募 30–50 名**非熟人**用户名单 + 邀请码机制

## D. 后端 · API ⬅ C

- [ ] P0 `POST /resumes`（上传/粘贴 → job(ingest)，入队前预占额度）
- [ ] P0 `GET /jobs/{id}`（`202 + Location`，阶段/心跳，可刷新恢复，指数退避 + `Retry-After`）
- [ ] P0 `PATCH /resumes/{id}`（字段纠错 → 新 revision，乐观锁）
- [ ] P0 `POST /resumes/{id}/match {jd}` → job(jd_match)
- [ ] P0 `POST /resumes/{id}/improve {jd}` → job(improve)，返回 diff
- [ ] P0 `POST /resumes/{id}/accept {patches}` → 新 revision
- [ ] P0 `POST /resumes/{id}/render` → job(render)（绑定已接受 revision + 模板版本）
- [ ] P0 `DELETE /resumes/{id}` → 触发删除状态机
- [ ] P0 **对象级授权**：从会话取 user，绝不信客户端 `user_id`

## E. 存储与 PII（🔴 合规可落地）⬅ A(数据流清单)

- [ ] P0 PostgreSQL 表：users/resumes/jobs/jd_matches/payments/entitlements/audit_log
- [ ] P0 **信封加密**：每份简历独立 DEK，KMS 管 KEK，应用不持长期主密钥
- [ ] P0 简历 **不可变 revision**；job 固定引用输入版本
- [ ] P0 **删除状态机**：冻结→删记录→删对象及历史版本→删临时产物→**销毁 DEK**→记证据→定期对账
- [ ] P0 备份策略：墓碑重放 + 明确过期；不可即删期间停止处理
- [ ] P0 日志**禁记**原文/Prompt/模型响应/OSS URL/手机号
- [ ] P0 `audit_log` 仅追加 + 完整性校验 + 独立保留期（actor/role/resource+version/purpose/request_id/result）
- [ ] P0 OSS 私有桶、禁列举、服务端签名 URL（绑定对象 + 短 TTL + `no-store`）

## 关键路径与排期判断

```
长板（决定日历时间，必须 W0 起并行跑）：
  ① ICP 备案（~20 工作日） ② PIPIA/等保/AI标识/法律意见 ③ 用户访谈定垂类+渠道

编码关键链（约 6–10 周，1–2 全栈）：
  B 引擎适配 → C 队列/隔离 → D API → (E 存储/PII ∥ F 沙箱) → G 前端 → H 支付 → I 风控 → K 演练

最易被低估、最不能砍：C 队列租约幂等 · E 删除闭环 · F 沙箱 · G 字段纠错+逐条diff
```

**判据**：A 的合规长板 + B 的真实质量回归先行；K 全过才邀请用户。预计**编码 6–10 周 + 合规并行 3–4 周起**，1–2 全栈约 **2–3 个月**到可上线的合规付费试点。
