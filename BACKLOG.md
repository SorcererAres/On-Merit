## XB. 海外合规与数据最小化（方案 B 新增，🔴 L0，需法律意见）

> 第二轮 Codex 复核结论：香港是技术节点非合规策略；核心是**去标识化后才出境/进模型**。

- [ ] `XB-1` 🔴 **PII 最小化层**（架构核心）：进 LLM/出境前检测剥离 姓名/电话/邮箱/地址/证件/照片/**第三方联系人**；只让去标识经历文本跨境，标识符不出境层重组；敏感信息命中即拦
- [ ] `XB-2` 🔴 **LLM 六项核实**：支持地区 + 企业合同 + DPA + 保留期 + 子处理者 + 删除 API（Gemini 地区出局、Claude 默认留 ~30 天 → 须零保留协议或换模型；禁个人账号/免费版）
- [ ] `XB-3` 🔴 **出境单独同意** + 告知接收方/目的/类型/境外保存；做 PIPIA 留档（≥3 年）；按量级定出境机制（小规模普通信息通常豁免评估/标准合同，但不免告知/最小化/权利）
- [ ] `XB-4` 🔴 **境内代表/机构**（PIPL §53）；**运营实质图**（签约主体/员工/域名/服务器/客服/收款/IP 各在哪），避免"香港名义、大陆实质"
- [ ] `XB-5` **生成式 AI 适用性**（暂行办法，向境内公众即适用，邀请码不豁免）：用户协议/输入保护/投诉举报/违法内容处置/模型备案适用性论证
- [ ] `XB-6` 第三方资源**服务端代理/自托管**（不用 Google Fonts/reCAPTCHA/海外存储直连）；跨境连通性按三大运营商/多省持续探测；不承诺"国内稳定可用"
- [ ] `XB-7` 香港 PDPO 义务；简历内**他人信息**（推荐人/证明人）自动检测删除，界面禁传他人资料
- [ ] `XB-8` PIPL 核心义务：隐私政策、用户权利闭环（`G-3`）、删除闭环（`G-2`）；**免费期禁广告画像/招聘导流/数据出售**（否则重构成经营性 → 方案 C）

> **首选验证起步**：客户端本地处理 / Concierge（服务器不持可识别简历）；或**双层架构**（境内主体存原件+境内脱敏，海外仅收去标识文本）。
